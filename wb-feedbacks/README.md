# Отзывы WB — AI-ответы

Отдельный минимальный сервис для ответов на отзывы Wildberries с AI-черновиками (YandexGPT). Не связан с [Юниткой WB](https://wb-unitka.vercel.app) — свой токен и свой деплой, без конфликтов лимитов API.

**Продакшен:** https://wb-feedbacks.vercel.app

## Локальный запуск

```bash
# из корня репозитория
npm install
npm run dev:feedbacks
```

Откроется http://127.0.0.1:5175

## Токен WB

1. ЛК WB → Профиль → Настройки → **Доступ к API**
2. Создайте токен с категориями:

| Категория | Обязательность | Для чего |
|-----------|----------------|----------|
| **Вопросы и отзывы** | обязательно | Список и ответы на отзывы |
| **Контент** | обязательно | Описание и характеристики для AI |
| **Цены и скидки** | рекомендуется | Премиум-апселл с ценой |
| **Статистика** | опционально | Сводный рейтинг SKU |

3. Вставьте токен на странице — он сохранится в **localStorage** этого браузера.

### Токен для коллеги

Токен **не синхронизируется** между людьми автоматически (в отличие от облака в Юнитке). Варианты:

1. **Ссылка с токеном** — после сохранения токена нажмите «Скопировать ссылку для коллеги». Токен передаётся в фрагменте URL (`#token=…`), не попадает в логи сервера. Отправьте ссылку в личку; коллега откроет — токен подставится и исчезнет из адресной строки.
2. **Свой токен** — каждый создаёт токен WB в ЛК (категория «Вопросы и отзывы») и вставляет у себя.

> Не публикуйте ссылку с токеном в общих чатах — это полный доступ к отзывам кабинета.

### Обновление версии в браузере

Если в подвале старый `commit`, а на сервере уже новый:

- Нажмите баннер **«Обновить сейчас»** (появится при рассинхроне с `/api/feedbacks/version`).
- Или жёсткое обновление: **Cmd+Shift+R** (Mac) / **Ctrl+Shift+R** (Windows).
- Проверка: `curl -s https://wb-feedbacks.vercel.app/api/feedbacks/version` — поле `commitSha` должно совпадать с `commit` в подвале страницы.

## YandexGPT на Vercel

1. [console.yandex.cloud](https://console.yandex.cloud/) → каталог (folder)
2. **Foundation Models** → API-ключ (тип Api-Key)
3. В Vercel → Settings → Environment Variables:

| Переменная | Описание |
|------------|----------|
| `YANDEX_GPT_API_KEY` | API-ключ Yandex Cloud |
| `YANDEX_CLOUD_API_KEY` | альтернативное имя для `YANDEX_GPT_API_KEY` |
| `YANDEX_FOLDER_ID` | ID каталога (`b1g…`) |
| `YANDEX_GPT_MODEL` | опционально: `yandexgpt-lite` (по умолчанию) |

Роль сервисного аккаунта: `ai.languageModels.user`. После добавления переменных — redeploy.

Опционально: `OPENAI_API_KEY` — запасной вариант, если YandexGPT недоступен.

## Автоответчик (10 отзывов/час)

Автоматическая отправка ответов с проверкой **manager-v9** (обесценивание, бан-фразы, качество).

### Клиентский режим (рекомендуется)

1. Вставьте токен WB и убедитесь, что YandexGPT настроен на Vercel.
2. Откройте блок **«Автоответчик»** под списком отзывов.
3. Включите переключатель **Автоответчик** — состояние сохраняется в `localStorage`.
4. **Держите вкладку открытой** — планировщик в браузере обрабатывает до **10 отзывов в час** (~1 каждые 6 мин).

В журнале: отправленные ответы, пропуски (валидация не прошла) и ошибки.

### Серверный cron (опционально, 24/7)

Если вкладку нельзя держать открытой:

| Переменная | Описание |
|------------|----------|
| `WB_API_TOKEN` | Токен WB продавца (категория «Вопросы и отзывы») |
| `CRON_SECRET` | Секрет Vercel Cron (автоматически при включении cron) |

Cron: `*/6 * * * *` → `GET /api/cron/auto-reply` — один отзыв за запуск.

## Деплой

Отдельный Vercel-проект **wb-feedbacks**.

**Продакшен:** https://wb-feedbacks.vercel.app (после импорта и деплоя)

### Импорт в Vercel (≈3 шага)

1. [vercel.com/new](https://vercel.com/new) → **Import** репозитория `egmen1-dev/wb-feedbacks`
2. **Project Name:** `wb-feedbacks` · **Branch:** `main` · **Root Directory:**
   - **рекомендуется:** оставьте пустым (корень репо) — подхватится корневой `vercel.json`
   - **или** укажите `wb-feedbacks` — подхватится `wb-feedbacks/vercel.json` (API всё равно из `api/feedbacks/*` в корне)
3. **Deploy** → затем **Settings → Environment Variables** (см. таблицу YandexGPT выше) → **Redeploy**

> Если страница пустая: в Vercel → **Deployments** → последний деплой → **Building** / **Logs**. Частая причина — неверный **Root Directory** или сборка не создала `wb-feedbacks/dist` (должны быть `index.html` и `assets/*.js`).

Конфиг сборки: на ветке `wb-feedbacks` в корне лежит `vercel.json`; на других ветках — `vercel.feedbacks.json` (CLI: `npm run deploy:feedbacks`).

CI (опционально): `.github/workflows/deploy-wb-feedbacks.yml` — секреты `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_FEEDBACKS_PROJECT_ID`.
